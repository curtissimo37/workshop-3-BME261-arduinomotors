# workshop-3-BME261-arduinomotors
BME261 motor workshop tracking changes to the arduino code from the oroginal motordrive code 
## Part 2 of Workshop 4: Application to your project  
> Objective: Configure a version control system for either a personal project or your group
project. You should start with your term project, but if your group all want to do this
workshop you may need to use files other than your group’s work

>Github
>1. Create a new repo on Github, in this repo you may store your group’s Arduino code,
or code from one of the previous activities. This may be public or private, but if it’s
private you must share it with me (cyoung@uwaterloo.ca).
>2. Update and commit changes to the code in your repository.
>3. Create a pull request with a change and pull it into your repository.
>4. There should be at least 3 commits in your branch.
>5. In the dropbox on LEARN submit a screenshot of the diff for the changes
incorporated via the pull request and a link to your repository.
